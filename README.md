# OLX Real Estate ETL Pipeline

## Опис проекту
Асинхронний ETL-пайплайн для збору, очищення та завантаження даних про нерухомість з платформи OLX у Google Sheets з урахуванням стійкості до змін верстки сайту та базового обходу систем захисту від ботів.

## Технологічний стек
* **Python 3.11+**
* **Extract:** `Playwright` (асинхронний парсинг, імітація дій користувача).
* **Transform:** `Pandas`, `re` (очищення даних, регулярні вирази, обробка пропущених значень).
* **Load:** `gspread`, Google Sheets API (автоматизоване вивантаження в хмару).
* **Concurrency:** `asyncio` (паралельна обробка з контролем навантаження).

## Архітектурні рішення (Highlights)

1. **Контроль конкурентності (Concurrency Control):**
   Реалізовано асинхронний збір даних за допомогою `asyncio.gather`. Для уникнення блокувань від Cloudflare (помилка HTTP 429) використано `asyncio.Semaphore`, який обмежує кількість одночасно відкритих вкладок браузера. Це імітує поведінку реального користувача, зберігаючи відносно високу швидкість обробки. В умовах продакшену при підключенні Proxy Pool цей ліміт можна масштабувати.

2. **Стійкість до помилок (Graceful Degradation & Fallback):**
   Якщо скрипт не знаходить дані за основними HTML-локаторами - активується механізм "Text Mining": скрапер витягує весь неструктурований текст сторінки, а `transformer.py` за допомогою регулярних виразів відновлює критичні дані (ціну, площу, поверх, локацію). Для обробки кейсів A/B тестування дизайну та динамічних CSS-класів. 

3. **Data Quality & Schema Evolution:**
   * Очищення цін від пробілів та форматування у тип `float`.
   * Визначення валюти (UAH, USD, EUR) та винесення її в окрему колонку.
   * Очищення локації (видалення назв вулиць, залишення лише міста).
   * Безпечна інтеграція з Pandas DataFrame для фільтрації невалідних записів.
